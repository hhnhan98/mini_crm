### BUSINESS RULES

# Account (User)

- System Role: Admin | Teamlead | Member
- Project Role
- Permission theo từng role
- Ràng buộc

- Có 3 role:
- Admin có thể: tạo projecj / task / account, gán user vào project / task, xem tất cả project / task, thay đổi trạng thái (status) của project / task,...
- Teamlead có thể: tạo và quản lý project mình thuộc về, thêm/xóa thành viên (quản lý members), thay đổi trạng thái project / task, tạo task và chỉ định member thực hiện (member phải thuộc project đó)
- Member có thể: xem task được chỉ định, comment task thuộc về, xem project mình thuộc về, upload task attachment mình thuộc về,...

Ràng buộc:

- Email là duy nhất
- Mỗi account (user) có 1 role duy nhất (đang nghĩ tới việc 1 account có thể nằm trong nhiều project/task, ví dụ Teadlead project A nhưng lại là Member project B, mình cần cho lời khuyên)
- Không được xóa account đang thuộc project (kể cả Admin, thao tác xóa tài khoản chỉ chuyển trạng thái unavailable chưa xóa hẳn khỏi db)
